docker version
  • To know docker version.
    • docker info
  • To know all info about installed docker on the system.
    • docker container run -it --publish 80:80 nginx
  • Flag 'it' means iteractive mode.
  • 80:80 - first 80 is the port exposed to local mechine and second port 80 or whatever like 3306 in case of sql for dependent application.
  • So we are mappin the port here.
  • If nginx is not there, it dowloads from dockerhub a public repository of docker.
  • Now type localhost in your browser and u'll see nginxr running.
  • To stop the container press ctrl + c.
    • docker container ls
  • To know all running containers.
    • docker container ls -a
  • To know all the containers whether it is running or not.
    • Fun Fact:

    Every docker container has a name and unique id (if u do not give ur container a name it will choose some random name) example in our case for nginx has xenodochial_northcutt so first parth is a adjective and second part is a name of either company/CEO/Famous personality/hacker/scientist.

    docker container rm 6b6
  • To remove a container 6b6 is the first 3 character of id of our docker container and it deleted successfully it will return back the id or name.
    • docker images
  • To see all the images.
  • With above command we have deleted the container but not the image so will still NGINX image available
    • docker image rm 62d
  • To delete a image from a docker
    • docker pull nginx
  • To pull image from docker hub.
  • It just pulls but does not creates a container.
    • docker container run -d -p 8080:80 --name mynginx nginx
  • Flag -d stands for detach means its runs in the background.
  • 8080 is the port for exposing to local and 80 for the container.
  • Running in the background means u'll not see the container logs and ur terminal won't be clogged up
  • nginx is the server image
    • docker container run -d -p 8081:80 --name myapache httpd
  • To pull and run a apache container in the background.
  • Official name aache server is httpd that u'll find on https://hub.docker.com.
