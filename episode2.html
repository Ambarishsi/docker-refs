Episode - 2

docker container run -d -p 3306:3306 --name mysql --env MYSQL_ROOT_PASSWORD=123456 mysql
  • Pulling mysql image and running a container in detched mode.
  • By passing some environment variable.
    • docker exec -it mysql mysql -uroot -p
  • To get into mysql console.
    • docker container stop 4a1
  • To stop a container running in detched/background mode.
    • docker container rm 4a1
  • To delete a container.
    • docker container exec -it mynginx bash
  • To bash into the container.
    • cd usr/share/nginx/html
  • Serving folder/Web root of nginx.
  • Do 'ls' to see the available files and you can also 'cat' that file to the see the content.
    • exit
  • To get out of the container.
    • docker rm $(docker ps -aq)
  • This remove all the container.
  • '-f' flag will remove even if the running container.
    • docker image build -t 9835625887/nginx-website .
  • For building ur images.
  • '-t' is a tag to recognise ur image.
  • '9835625887' is ur docker hub account id.
  • 'nginx-website' is the name of the image.
    • docker image build -t test-locally .
  • If u don't wish to push ur images to docker hub.
    • docker container run -d -p 9001:80 test-locallly
  • To run ur image on the docker container.
    • docker push 9835625887/nginx-website
  • To push ur image to docker hub, so u can access it form anywhere in the world.
    • docker login
  • In case terminal asks u to authenticate.
    • Dockerfile command:

    FROM nginx:latest
    WORKDIR /usr/share/nginx/html
